A Simple Processor Pipeline

The simplest pipeline - hardly worth the name! - will have two stages. In a processor these will be a prefetch stage which fetches an instruction and an execution stage which does the processing work.

A slightly more illustrative pipeline is exemplified by the early (and some current) ARM implementations with three stages.

Fetch
Read an instruction as indicated by PC, including instruction memory access; increment PC.
Decode
Work out what the instruction is to do.
Execute
Read register operands, perform any calculation (including any address derivation) and - unless the operation is load or store - write the result back.

An advantage of this structure is that it avoids many hazards in the execution phase. A disadvantage of this structure is that many functions are crammed into the 'execute' stage, slowing it, and hence the whole pipeline, down.
This was also intended to facilitate interspersing load and store execution with instruction fetches with a single memory interface. Note how the memory is shared but kept busy under (nearly) all circumstances.
